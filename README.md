# Android Studio
<b>Позволяет создавать нативные приложения для Android OS<br>Android - открытая операционная система, то есть свободно модифицируемая</b>

## Устройство платформы Android
<b>С точки зрения архитектуры Android можно выделить следующие уровни:
1) Базовый уровень(linux Kernel) - уровень абстракции между аппаратным уровнем и программами пользователя
2) Набор библиотек и среда использования(Libraries and Android Runtime) обеспечивает важнейшую базовую функциональность для приложения, содержит виртуальную машину Dalvik и базовые библиотеки Java, необходимые для запуска Android приложений
3) Уровень каркаса приложений(Application Framework) обеспечивает разработчикам доступ к API, предоставляемым компонентами системы уровней библиотек
4) Уровень приложения (Application) - набор предустановленных базовых приложений.</b><br>

### Все Android-приложения имеют расширения .apk(Android package)

<b>JDK</b> - бесплатно распространяемый комплект разработчика приложений на языке Java, включающий в себя компилятор Java, стандартные библиотеки классов Java, примеры, документацию, различные утилит ыи исполнительную систему Java Runtime Environment. В состав JDK не входит интегрированная среда разработки(IDE)


<b>Android Software Development Kit(SDK)</b> - содержит множество инструментов и утилит для создания и тестирования приложений

## API
<b>Интерфейс, который обеспечивает связь между двумя разными "частями" программного обеспечения</b>

## Инструменты для разработки приложений под Android
<b>SDK Manager</b> - инструмент, позволяющий загрузить компоненты Android SDK. <br>
<b>Android Emulator</b> - виртуальное мобильное устройство, которое создаётся и работает на компьютере разработчика, используется для разработки и тестирования мобильных приложений без привлечения реальных устройств<br>
<b>AVD Manager</b> - предоставляет графический интерфейс для создания виртуальных Android устройств (AVDs), предусмотренных Android Emulator, и управления ими <br>
<b>Android Debug Bridge</b> - гибкий инструмент, позволяющий управлять состоянием эмулятора или реального Android устройства, подключённого к компьютеру<br>

## Структура Android-приложений
 - **Gen файлы**, сгенерированные самой Java(обязательно)
 - **AndroidManifest.xml** - файл манифеста предоставляет системе основную информацию о программе(обязательно)
 - **Src** - каталог, в котором содержится исходный код приложения(обязательно)
 - **Assets** - произвольное собрание каталогов и файлов(не обязательно)
 - **Res** - каталог, содержащий ресурсы приложения(обязательно)
